### **1.2 产品目标**

- **规范化**：固化“统一归口、两级集中”的管理机制，确保所有服务商管理流程符合集团制度。
- **集约化**：建立统一的服务商资源库，避免重复建设和信息孤岛，实现资源全局共享。
- **精益化**：通过“五维一图”服务商画像和量化考核，为选商和决策提供精准数据支撑。
- **协同化**：连接集团总部、专业公司、所属单位及服务商，实现多方高效线上协同。
- **智慧化**：通过动态监控和风险预警，实现从被动响应到主动风险防控的转变。

### **1.3 目标用户**

- **集团管理者**：工程和物装管理部、总部相关部门。关注全局资源状况、战略服务商发展、风险监控。
- **所属单位用户**：各专业公司、所属单位的服务采购方、归口管理部门、管理小组成员。负责具体的服务商寻源、使用、考核、日常管理。
- **服务商**：注册及潜在服务商。在线提交资料、查询状态、接收通知、参与考核。
- **系统管理员**：负责系统配置、权限分配、字典维护。

## **4. 功能模块详细设计**

### **4.1 服务商全生命周期管理模块**

#### **4.1.1 服务商注册、实名认证与准入**

- **功能描述**：支持个人/企业联系人注册个人账号，完成实名认证后获取“服务商标签”，待采购中选后再发起正式准入申请，经审核和MDG同步后成为合格服务商。
- **用户角色**：潜在服务商（个人/企业联系人）、归口管理部门、采购中心、管理小组。
- **前置条件**：系统需支持手机号/邮箱注册，支持实名认证接口（如公安、工商库）。
- **详细流程与细节**：
  **第一步：注册个人账号**
  - 用户通过手机号或邮箱注册，设置密码，完成基础账号创建。
  - 填写个人基本信息：姓名、身份证号、联系电话等。
  **第二步：完成个人实名认证**
  - 系统对接第三方实名认证接口（如支付宝、微信、或公安部接口）。
  - 用户上传身份证正反面照片，进行人脸识别或银行卡验证。
  - 认证通过后，账号状态变为“已实名个人用户”。
  **第三步：获取服务商标签**
  - 实名认证通过后，用户可申请“服务商标签”。系统允许同一用户申请多个服务商标签（代表不同服务主体）。
  - 用户选择服务商类型（**企业、其他组织、自然人**），填写基础信息（名称、统一社会信用代码/身份证号等）。系统进行基础查重。
  - 生成一个“服务商标签”实体，与个人账号关联。此时服务商状态为“**潜在服务商**”，尚未获得准入资格。
  **第四步：参与采购并中选**
  - 潜在服务商可凭借服务商标签参与集团组织的服务采购活动（如询价、招标）。
  - 中选后，系统自动向服务商联系人发送通知，提示“请完成正式准入申请”。
  **第五步：发起服务商准入申请**
  - 服务商登录个人账号，进入“服务商准入”模块，选择对应的“服务商标签”和中选项目。
  - 根据服务商类型（企业/其他组织/自然人），填写差异化的详细资料并上传资质文件（详见下文 **服务商类型与资质要求**）。
  - 提交准入申请，进入管理单位审核流程。
  **第六步：管理单位审核**
  - 归口管理部门或采购方组织审核，可分配管理小组进行评审、现场考察等。
  - 审核意见：通过、驳回（需说明理由）、要求补充材料。
  **第七步：同步MDG获取往来单位编码**
  - 审核通过后，系统自动调用集团公司MDG（主数据管理）系统接口，申请生成“往来单位编码”。
  - 接口需传递：服务商名称、统一社会信用代码（自然人传身份证）、服务商类型、银行账户等信息。
  - 系统支持手动重试和失败告警。
  **第八步：MDG返回编码，准入成功**
  - MDG系统返回唯一编码后，系统将该编码关联到服务商标签，服务商状态变更为“**合格服务商**”，正式纳入服务商资源库。
  - 若MDG返回失败或拒绝，系统标记准入失败，通知管理员人工介入处理。

#### **4.1.2 服务商类型与资质文件要求**

> 根据办法第十六条及实际业务需求，不同类型服务商在准入时需要填写的信息和提交的资质文件存在差异。系统需支持动态表单配置。


| **信息/文件类别**    | **企业**                                   | **其他组织（如事业单位、社会团体、合伙企业等）**               | **自然人**                      |
| -------------- | ---------------------------------------- | ---------------------------------------- | ---------------------------- |
| **基础信息字段**     | 企业名称、统一社会信用代码、法定代表人、注册资本、成立日期、注册地址、经营范围等 | 组织名称、统一社会信用代码（或登记证号）、负责人、注册资金、组织类型、业务范围等 | 姓名、身份证号、性别、出生日期、联系地址         |
| **证照文件**       | 营业执照（正本/副本） 开户许可证 一般纳税人资格证明（可选）          | 法人登记证书 / 社会团体登记证 / 合伙企业营业执照等 银行开户证明      | 身份证正反面扫描件 个人银行卡照片（用于收款）      |
| **财务状况文件**     | 近三年审计报告（或财务报表） 新成立企业按年限提供                | 近三年财务报告或上级单位出具的财务证明 或银行出具的资信证明           | 个人银行流水（近6个月）或存款证明 或收入证明      |
| **合规信用文件**     | 无重大违法违规承诺书 信用中国、国家企业信用信息公示系统截图 无行贿犯罪记录证明 | 类似的合规承诺及查询截图                             | 个人无犯罪记录证明 未被列入失信被执行人名单（查询截图） |
| **QHSE/ESG文件** | 质量管理体系认证（可选） 安全许可证（如需要） 环境管理体系认证 社保缴纳证明  | 根据行业特点提供相应安全、质量证明                        | 无相关要求，但需签署安全环保承诺书            |
| **人员资质（如适用）**  | 关键岗位人员名单、资格证、劳动合同                        | 类似的人员资质                                  | 个人从业资格证（如律师证、咨询师证等）          |
| **其他特殊文件**     | 经营场所证明（租赁合同或产权证） 授权委托书（如非法人办理）           | 根据组织类型补充                                 | 无                            |


> **产品设计要点**：
>
> - 使用动态表单，用户选择服务商类型后，展示对应的字段和文件上传项。
> - 支持文件格式校验（PDF/JPG/PNG，单文件<20M）。
> - 部分文件（如营业执照）支持OCR识别，自动填充基础字段，减少手工录入。

#### **4.1.3 服务商信息变更管理**

- **功能描述**：服务商准入成功后，其基本信息、资质文件、人员等信息发生变化时，支持在线提交变更申请，并经过审核后生效，全程留痕。
- **用户角色**：服务商联系人、管理单位审核人员。
- **适用变更类型**：
  - **基础信息变更**：单位名称、法定代表人/负责人、注册地址、经营范围、联系方式等。
  - **资质文件变更**：营业执照更新、资质证书延期/升级、银行账户变更等。
  - **关键人员变更**：项目经理、安全负责人、技术负责人等。
  - **服务范围变更**：增加或减少服务分类代码。
- **流程与细节**：
  1. **发起变更申请**：
    - 服务商登录系统，进入“服务商中心” -> “信息变更”。
    - 选择变更类型，修改相应字段或重新上传文件。
    - 系统高亮显示变更前后差异，便于审核人员识别。
    - 提交申请，生成变更单号。
  2. **变更审核**：
    - 根据变更内容的敏感程度，系统可配置不同的审核流程：
      - **简易变更**（如联系电话、邮箱）：自动审核通过，或仅需采购方一级确认。
      - **重要变更**（如单位名称、法定代表人、银行账户）：需提交至原准入管理单位，进行类似准入的审核流程，可能涉及现场复核。
      - **关键人员变更**：需经对应服务采购方批准，确保替代人员满足项目要求（对应办法第二十四条）。
    - 审核人员可对比变更前后信息，查看上传的新文件。
    - 审核结果：通过、驳回（需注明理由）、要求补充材料。
  3. **审核生效与同步**：
    - 审核通过后，系统自动更新服务商主数据信息。
    - 若变更涉及MDG关键字段（如单位名称、银行账户），系统需再次调用MDG接口进行更新，获取新的往来单位编码或确认同步成功。
    - 更新失败时，系统冻结变更生效，并通知管理员。
  4. **变更历史追溯**：
    - 所有变更记录（包括申请时间、变更内容、审核意见、生效时间）永久保存。
    - 在服务商画像中提供“变更履历”标签页，支持导出。

### **4.2 服务商考核评价模块**

- **功能描述**：量化考核（一单一评/年度考评）和问题事故事件记分，并生成最终考核等级。
- **用户角色**：服务采购方、管理小组、服务商（查看结果）。
- **流程与细节**：
  1. **量化考核指标库**：
    - 系统提供统一的基础考核指标模板（合同执行、服务响应、质量、交付、满意度等）。
    - 各所属单位可在基础模板上，细化本单位考核内容。
  2. **一单一评**：
    - **触发**：合同验收或结束时。
    - **执行**：采购方/最终用户在项目验收单旁边完成电子评价表，打分并填写评语。
    - **流程**：评价提交 -> 服务商可查看 -> 有异议可发起复核。
  3. **年度考评**：
    - **触发**：每年第一季度。
    - **执行**：系统自动聚合上年度该服务商在本单位的所有“一单一评”分数、履约数据，形成基础分。考核人进行人工调整和综合评价。
    - **流程**：系统计算预评分 -> 考核人确认/调整 -> 审批 -> 结果公示 -> 服务商可查看。
  4. **问题事件记分**：
    - 建立“问题事件库”（附录A），预设扣分标准和对应分值（如：一般安全事故扣X分）。
    - 当发生质量、安全、合规等事件时，由采购方或监督部门在系统登记事件，上传证据，系统自动按规则扣除相应分数。
  5. **综合评级与发布**：
    - 系统自动计算综合得分 = (量化考核分) - (问题事件扣分) -> 总分映射为A/B/C/D级。
    - 自动控制A类≤20%，C类在10%-20%之间。
    - 考核结果在平台公开发布（服务商可见自身及等级），同时作为采购模块的选商依据。

### **4.3 服务商日常监管与画像模块**

- **功能描述**：动态监控服务商状态，构建“五维一图”服务商画像，并提供风险预警。
- **用户角色**：所有管理角色、服务商。
- **流程与细节**：
  1. **服务商画像**：
    - **数据汇聚**：从注册、合同、考核、事件等模块自动、实时抽取数据，填充“五维”（基本信息、业务能力、履约数据、合规信用、QHSE/ESG）。
    - **可视化展示**：使用雷达图、仪表盘等形式，直观展示该服务商在各维度的得分及与行业平均水平的对比。
  2. **信息维护与变更**：（已纳入4.1.3）
  3. **动态风险监控**：
    - 系统定时或实时轮询：
      - 内部数据：资质失效日期、考核结果为D、累计扣分超阈值。
      - 外部数据：（通过API对接）信用中国、国家企业信用信息公示系统、最高法院失信名单。
    - **预警规则**：当触发预设风险指标时（如“信用中国”列入严重失信主体），系统自动：
      - 生成高亮风险提示，推送至相关采购方和管理部门。
      - 触发人工复核流程，指派专人处理。
      - 服务商画像状态自动更新为“高风险”。

### **4.4 战略服务商管理模块**

- **功能描述**：支持战略服务商的遴选、协议签订、关系维护及特殊策略执行。
- **用户角色**：集团工程物装部、相关专业公司。
- **流程与细节**：
  1. **战略候选人生成**：
    - 针对特定服务品类，支持在线发布公开招标资格预审公告。
    - 线上组织评审，形成候选战略服务商名单。
  2. **谈判与签约**：
    - 对候选人，组织跨部门（采购、技术、业务）线上/线下谈判，记录谈判纪要。
    - 在线拟定《战略采购协议》，提交“集团物装供应链管理委员会”线上审定。
    - 协议签署后，在系统中给相关服务商打上“战略服务商”标签。
  3. **战略关系管理**：
    - 为战略服务商设定权限，使其在采购模块能看到专属的优先项目或框架协议。
    - 建立“资源互供清单”功能模块，记录双方互相提供的资源与优惠政策。
    - 一旦战略服务商考核为C/D级或有重大违约，采购方可在系统内发起“取消战略资格”流程。

### **4.5 退出管理模块（含手动冻结/解冻）**

- **功能描述**：管理服务商的退出机制，包括自动触发暂停/取消、**手动冻结/解冻**、以及恢复流程。
- **用户角色**：各级采购方管理员、集团归口管理部门、监督部门。
- **流程与细节**：

#### **4.5.1 自动退出处理（依据办法第五十五条至五十九条）**

- 考核为D类、发生一般事故、单方解除合同等，系统自动触发暂停服务资格（暂停期半年）。
- 触发“三商”黑名单情形的，自动取消服务资格。

#### **4.5.2 手动冻结与解冻**

- **应用场景**：
  - 管理人员发现服务商存在疑似违规行为，但尚在调查中，需要临时冻结其参与新采购的权限。
  - 服务商主动申请暂停业务（如业务调整、资质升级期间）。
  - 系统与MDG同步失败或服务商在MDG中被锁定，需手动同步状态。
- **操作权限**：需经所属单位归口管理部门负责人审批，涉及集团范围的需集团工程物装部审批。
- **操作流程**：
  1. **发起冻结申请**：
    - 管理员在服务商详情页点击“手动冻结”。
    - 填写冻结原因、冻结范围（集团/专业/企业级）、预计冻结时长（可规定最长不超过6个月）。
    - 上传相关证据或说明文件。
  2. **审批流程**：
    - 根据冻结范围，系统路由至对应级别的审批人。
    - 审批通过后，服务商状态立即变为“**手动冻结**”。
  3. **冻结期管理**：
    - 冻结期间，该服务商不得参与任何新的服务采购活动（包括询价、投标、谈判）。
    - 已有的在建合同可继续履行（或视合同约定），系统应予以标识但不强制中断。
    - 服务商画像中展示“冻结中”标签及原因。
  4. **解冻操作**：
    - **手动解冻**：问题调查结束后，管理员可发起“解冻申请”，填写处理结论，经审批后恢复服务商正常状态。
    - **自动解冻**：如果设置了预计冻结时长，到期后系统可自动解冻，或提醒管理员进行复核后解冻（可配置）。
  5. **审计日志**：所有的冻结、解冻操作均记录操作人、时间、原因、审批意见。

#### **4.5.3 暂停与恢复补充**

- 原办法中的“暂停资格”自动触发后，系统同样提供“手动恢复”入口，支持管理员在整改审核通过后提前恢复（不必等180天期满），但需上级审批。

#### **4.5.4 黑名单管理**

- 取消资格后，服务商进入“黑名单”状态，支持手动拉出黑名单（重新准入），但需走特殊审批流程。

### **4.6 分包商管理模块**

- **功能描述**：对服务商所使用的外部分包商进行线上管理，实现分包商信息登记、与主服务商关系绑定、采购方审批、履约监督等全流程支持。
- **用户角色**：主服务商、服务采购方、管理单位。
- **业务规则**（依据办法第二十九、三十条）：
  - 服务商如需分包，必须事先在系统中维护拟分包商名单及分包范围。
  - 分包商名单需提交服务采购方认可（审批），未经认可不得擅自启用。
  - 分包活动需签订分包合同，并上传到系统备案。
  - 主服务商对分包商负全过程管理责任，采购方有权监督。
- **详细流程与细节**：
  #### **4.6.1 分包商注册（成为分包商）**
  - **方式一：独立注册为服务商**：任何法人或自然人可以按照4.1.1流程，成为集团合格服务商。之后可以被其他主服务商选择为分包商。
  - **方式二：仅作为分包商注册（轻量级）**：
    - 主服务商在系统中发起“添加分包商”，填写分包商基本信息（名称、联系人、资质等），上传营业执照/身份证等文件。
    - 系统不要求该分包商走完整准入流程，但需生成一个“分包商”类型的轻量级账号（不可直接与集团交易）。
    - 采购方审批通过后，该分包商即可用于该项目。
  - **建议**：对于长期合作或资质要求高的分包商，采用方式一；对于临时、低风险的分包商，采用方式二。
  #### **4.6.2 分包关系绑定**
  - 主服务商在某个具体项目/合同下，发起“分包商绑定申请”。
  - 需填写：分包商名称（从已注册库中选择或临时添加）、分包工作内容、分包金额（可选）、分包合同期限、拟签订的分包合同扫描件。
  - 提交至服务采购方（或项目管理方）审批。
  - 审批通过后，系统建立“主服务商-分包商-项目”的关联关系。
  #### **4.6.3 分包商履约监督与考核**
  - 服务采购方可以在系统中记录对分包商的检查情况、问题事件。
  - 分包商的问题会关联到主服务商的考核（根据办法第三十条）。
  - 系统支持生成分包商黑名单，如果某一分包商被多次违规记录，可被禁止在所有项目中作为分包商使用。
  #### **4.6.4 分包商信息变更与退出**
  - 分包商的基本信息变更可由主服务商代为提交，或分包商自行登录轻量账号修改（如有）。
  - 分包关系解除：主服务商可在系统申请“解除分包”，经采购方确认后，关系失效。
  #### **4.6.5 数据展示**
  - 在主服务商详情页，展示“分包商列表”，包括每个分包商绑定的项目、审批状态。
  - 在采购方视角，可查看所有分包商及关联的主服务商，便于风险穿透管理。

### **4.7 组织与权限管理模块**

- **功能描述**：支撑“统一归口、两级集中”的管理机制。
- **用户角色**：系统管理员。
- **流程与细节**：
  1. **组织架构管理**：
    - 支持树形结构配置集团总部、专业公司、所属单位等。
  2. **角色与权限**：
    - 预置角色：集团归口管理员、专业公司管理员、所属单位采购员、所属单位考核员、监督员、管理小组成员（可动态创建小组）。
    - 权限控制数据行（只能看本单位/本专业数据）和列（如：财务人员只能看财务相关字段）。
  3. **管理小组**：
    - 允许采购项目的创建者，从组织架构中选择人员，临时或长期组建一个“管理小组”。
    - 小组长有权限分配小组内的寻源、考核、检查等具体任务。

## **6. 非功能性需求**

- **性能要求**：
  - 服务商画像、仪表盘等关键查询页面加载时间 < 3秒。
  - 支持至少注册服务商数量为5万家。
  - 外部接口数据同步（如信用中国）每日定时更新，延迟 < 24小时。
  - 分包商模块需支持每个主服务商管理不超过500个分包商关系。
- **可用性**：
  - 系统7x24小时可用，计划内停机需提前通知。
  - Web端应适配主流浏览器（Chrome, Edge, 360等）。建议开发移动端H5或小程序，用于现场考察和移动审批。
- **安全性**：
  - 严格的数据权限隔离，确保所属单位只能访问本单位数据。
  - 服务商与外网用户交互数据需加密传输。
  - 关键操作（如考核评分、暂停准入）需保留完整的审计日志。
- **集成性**：
  - 需与集团采购系统（寻源、合同）、财务系统、主数据管理平台（MDG）、外部征信数据平台（信用中国等）进行API集成。
