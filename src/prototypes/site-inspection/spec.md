# 现场考察管理

## 📋 业务与功能

### 1.1 核心目标

现场考察管理页面用于对供应商/合作伙伴的现场考察申请进行审批管理，支持按状态（待审批/已通过/已拒绝）分类查看、详情查阅及在线审批操作。

### 1.2 功能清单

- **Tab 分类切换**：待审批 / 审批通过 / 审批拒绝 三个分类切换
- **搜索与筛选**：按编号/名称搜索，按类型（现场考察/视频考察）、状态筛选
- **数据表格**：展示编号、名称、类型、状态、创建人、创建时间、操作列
- **查看详情弹窗**：展示完整考察信息（含地址、日期、负责人、描述、审批记录）
- **审批弹窗**：审批人填写意见后通过/拒绝，审批后自动刷新并切换 Tab
- **侧边菜单**：作为菜单项页面，与 SideMenu 集成

### 1.3 交互要点

- Tab 切换时过滤对应状态数据
- 搜索和筛选联动，实时过滤表格数据
- "查看"按钮弹出详情弹窗
- "审批"按钮弹出审批弹窗，提交后自动刷新列表并切换到对应 Tab
- 审批记录以时间线形式在查看弹窗中展示

## 📊 内容规划

### 2.1 信息架构

```
现场考察管理
├── 侧边菜单（系统导航）
└── 内容区
    ├── 页面头部（标题 + 搜索 + 筛选）
    ├── Tab 切换区（待审批 / 审批通过 / 审批拒绝）
    └── 数据表格
        └── 操作列
            ├── 查看（弹窗）
            └── 审批（弹窗）
```

### 2.2 数据来源

- **数据类型**：现场考察记录（`inspections`）
- **数据源**：内置示例数据
- **关键字段**：
  - `id`: 编号（如 SC-2024-001）
  - `name`: 考察名称
  - `type`: 类型（site=现场考察 / video=视频考察）
  - `status`: 状态（pending=待审批 / approved=审批通过 / rejected=审批拒绝）
  - `creator`: 创建人
  - `approver`: 审批人
  - `createTime`: 创建时间
  - `inspectDate`: 考察日期
  - `address`: 考察地址/地点
  - `manager`: 负责人
  - `contact`: 联系方式
  - `description`: 考察描述
  - `approvalRecords`: 审批记录列表（审批人、审批意见、决策、时间）

### 2.3 内容示例

**考察记录示例**：
- SC-2024-001 / 供应商生产车间考察 / 现场考察 / 待审批 / 张三 / 2024-06-01
- SC-2024-002 / 远程资质核验 / 视频考察 / 审批通过 / 李四 / 2024-06-05

## 🎨 布局与结构

### 3.1 整体布局

- **布局模式**：双栏（左侧固定导航 + 右侧内容区）
- **容器宽度**：流式
- **关键尺寸**：
  - 侧边菜单宽度：展开 `240px` / 折叠 `64px`
  - 页面头部分为标题区与操作区两行
  - 表格自适应宽度

### 3.2 响应式适配

- **桌面端（≥1200px）**：双栏布局，侧边菜单常驻展开
- **平板端（768-1199px）**：侧边菜单默认折叠
- **移动端（<768px）**：侧边菜单折叠，表格横向滚动

## ⚙️ Axure API 说明

### 5.1 事件列表（eventList）

- `onViewClick`：点击查看按钮时触发，携带当前记录数据
- `onApproveClick`：点击审批按钮时触发，携带当前记录数据
- `onApproveSubmit`：提交审批时触发，携带审批结果

### 5.2 动作列表（actionList）

- `refreshData`：刷新页面数据

### 5.3 变量列表（varList）

- `selected_inspection`：当前选中的考察记录
- `active_tab`：当前选中的 Tab 状态

**命名规范**：使用 snake_case（小写字母、数字、下划线）

### 5.4 配置项列表（configList）

- `title`：页面顶部标题，默认值为"现场考察管理"

### 5.5 数据项列表（dataList）

**数据结构**：

```typescript
{
  inspections: InspectionRecord[]
}
```
